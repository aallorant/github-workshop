# Organising and sharing your research code

A 90-minute hands-on workshop for doctoral students.

▶ **[Open the slides](https://aallorant.github.io/github-workshop/slides/slides.html)** · [source `.qmd`](slides/slides.qmd) · [profile template](templates/profile-readme-template.md) · [project template](templates/project-readme-template.md)

## By the end, you will have

1. A **GitHub profile** with a short personal README, suitable to put on your CV.
2. **One of your projects** organised as a public GitHub repository with a clear README, a key figure, and a short description of your methods and data.

No prior experience with Git or GitHub is required.

## Before you arrive

If you have time (5–10 min):

- Install **GitHub Desktop** from [desktop.github.com](https://desktop.github.com).
- Create a **GitHub account** at [github.com](https://github.com) (use your institutional email if you want to be eligible for [GitHub Education](https://education.github.com/) benefits).
- Sign into GitHub Desktop with your new account.

If you do not manage these before the session, we will do them together in the first 10 minutes.

**Also bring**:

- A folder for one of your current projects (a paper, dissertation chapter, or analysis).
- One figure (PNG or JPG) from that project that you would like to feature.
